## 已完成的可靠性加固（不改变产品 Phase 判定）

- 分片上传用户隔离、尺寸/序号校验、完成幂等、过期会话和临时文件清理。
- 对象存储/数据库/入队失败补偿：避免孤立 blob 或悬空录音；失败录音保留为 `error` 便于恢复。
- JSON 存储采用跨进程锁、遗留锁 TTL 回收、临时文件 fsync + 原子 rename。
- BullMQ 稳定 jobId、3 次指数退避、任务总时限和 stalled-job 恢复；内存队列去重。
- provider HTTP 请求统一超时；本地/S3 音频支持 Range/HEAD；共享链接密码使用随机盐 scrypt 并带失败限流。

## 仍需重点验证的生产风险

- S3/Redis/PostgreSQL 真实环境集成测试、故障恢复和孤儿 blob 扫描尚未完成。
- JSON 存储已有跨进程锁和轮换备份，生产环境仍强制使用 PostgreSQL；需补数据库备份/恢复演练。
- 任务级 AbortSignal 尚未贯穿所有 ASR/LLM provider，超时只能让 pipeline 尽快报错，不能取消所有底层请求。
- Flutter 静态分析/端到端测试需在稳定 Flutter 环境重试；工作区曾出现 analysis server 崩溃。
- 约 21GB 本地 ASR 模型的下载、构建缓存、真实音频和 `.asr.*` 文件保留/备份策略仍待明确。
